## Findings

### 🟠 HIGH — 2

#### H-1: `package-lock.json` tracked alongside `pnpm-lock.yaml`

- **File**: `package-lock.json` (224KB)
- **Issue**: Two lock files create ambiguity. `pnpm` is the canonical package manager; `package-lock.json` is stale and could confuse CI or contributors.
- **Impact**: Dependency resolution inconsistency if someone runs `npm install` instead of `pnpm install`.
- **Fix**: Remove `package-lock.json` from tracking + add to `.gitignore`.

#### H-2: Dependency vulnerabilities (pnpm audit)

- **Vulnerable packages**:
  - `vitest <3.2.6` — CRITICAL: arbitrary file read via Vitest UI server
  - `ws 8.20.1` — HIGH: memory exhaustion DoS
  - `undici 8.2.0` — HIGH: TLS validation bypass + WebSocket DoS (×3 advisories)
  - `hono <4.12.25` — HIGH: CORS middleware credential reflection
  - `protobufjs <=7.6.0` — HIGH: DoS via unbounded Any expansion
  - `vite 7.x` — HIGH: `server.fs.deny` bypass (Windows-only, low impact on Termux)
- **Impact**: Real for dev/test dependencies; lower for production (bundle doesn't ship `vitest`/`vite`).
- **Fix**: `pnpm update vitest ws undici` + verify `hono`/`protobufjs` are transitive (upstream update needed).

### 🟡 MEDIUM — 4

#### M-1: Non-Termux files still tracked

- `docs/windows/README.md` — Windows documentation irrelevant to Termux-only fork
- `package.json` scripts still include `build:binary-linux-x64`, `build:binary-windows-x64` — Bun binary targets that cannot run on Termux
- **Fix**: Remove `docs/windows/`, clean up non-Termux build scripts.

#### M-2: Launcher route 3 always exits 0

- **File**: `bin/kimchi:150`
- **Issue**: Long commands (`--print`, coding tasks) always `exit 0` regardless of the child process exit code. This masks auth failures, provider errors, and API 401s.
- **Impact**: Silent failures in non-interactive mode.
- **Fix**: Capture `$CHILD_PID` exit code via `wait` and propagate.

#### M-3: `src/extensions/activity.ts` hardcodes `/tmp/kimchi`

- **File**: `src/extensions/activity.ts:20`
- **Issue**: `const SOCKET_DIR = "/tmp/kimchi"` — Termux may not have `/tmp` in some environments.
- **Impact**: Socket creation fails silently in sandbox mode (graceful degradation, not a crash).
- **Fix**: Use `path.join(os.tmpdir(), "kimchi")` instead.

#### M-4: Bundle ships platform-specific clipboard loader code

- **Bundle reference**: `@mariozechner/clipboard-darwin-arm64`, `clipboard-win32-arm64-msvc`, etc.
- **Issue**: 9 platform-specific clipboard module references remain in the bundle (guarded by try/catch).
- **Impact**: Dead code; increases bundle size by ~1-2KB; no runtime risk.
- **Fix**: Add `@mariozechner/clipboard-*` to the external list in `build-bundle.mjs` to tree-shake them.

### 🟢 LOW — 3

#### L-1: Upstream URLs in source code

- `src/commands/update.ts:251` — references `getkimchi/kimchi` install script
- `src/extensions/report-bug.ts:13` — references `getkimchi/kimchi/issues/new`
- `src/config.ts:22-24` — references `llm.kimchi.dev` and `cast.ai` telemetry endpoints
- **Impact**: Cosmetic; these are upstream references that work as-is.

#### L-2: `TODO`/`FIXME` comments

- 41 matches across source. Key ones:
  - `src/models.ts:162` — TODO about thinking.type.enabled for Anthropic models
  - `src/extensions/permissions/index.ts:378` — FIXME about ferment session tracking
  - `src/extensions/model-catalog/builtin-models.ts:103` — TODO about model metadata API
- **Impact**: Known limitations, not bugs.

#### L-3: `config.json` security settings verified but drift possible

- Kimchi may rewrite `config.json` with `telemetry.enabled: true` after use.
- The launcher sets `KIMCHI_TELEMETRY_ENABLED=0` which overrides, but config file may show `true`.
- **Impact**: Low (env var takes precedence).
- **Fix**: Periodically run `verify_security_hardening.py` to catch drift.
